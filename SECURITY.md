# 安全策略

## 报告漏洞

如果你发现安全漏洞，请**不要**公开提交 Issue。

通过 [GitHub Security Advisory](https://github.com/ai-kangaroo/flowonce/security/advisories/new) 私下报告。

请包含：
- 漏洞描述
- 复现步骤
- 影响范围
- 建议修复方案（可选）

我们会在 **48 小时内**确认收到，并在 **7 天内**给出初步评估。

## 安全设计

FlowOnce 的安全设计是架构级的，不是附加的：

| 层级 | 机制 |
|------|------|
| 录制 | 全程本地，事件流不上传云端 |
| 授权 | 每次录制需明确同意，悬浮窗常驻 |
| 时限 | 单次录制上限 30 分钟 |
| 敏感值 | 编译期强制替换为占位符，无法写入技能包 |
| 生成 | 必须经过人工确认（`reviewed: true`）才允许生成技能 |
| 执行 | 高危操作（删除、发消息、财务）需二次确认 |

## 赏金

暂无赏金计划，但我们会在致谢页面感谢负责任的漏洞报告者。
